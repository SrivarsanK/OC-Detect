# Plan: 05-01 Cloud Synchronization Logic

## Requirements Covered
- **SYNC-01**: Secure cloud sync.
- **SYNC-02**: Connectivity retry logic.
- **SYNC-04**: Transit encryption.

## Goals
Implement a **SyncService** to securely "push" clinical bundles to a central repository.

## Dependencies (Wave 0 - Parallelizable)
- `httpx` (Async requests)
- `tenacity` (Retry logic)

## Tasks (Wave 1)

1. **Case Bundler (Async)**
    - Implement `SyncService.bundle_case(case_id: str)`
    - Package: Enhanced image, Heatmap image, JSON Report, PDF Report.
2. **Sync Client (httpx)**
    - Implement `SyncService.push_case(case_id: str)`
    - Handle multi-part form-data uploads.
    - Status reporting: Transitions Case from `PROCESSED` → `SYNCED`.
3. **Retry Engine (Tenacity)**
    - Implement exponential backoff for failed uploads.
    - Max retries: 5, with initial wait of 2s.
4. **Integration with Case History**
    - Provide a manual `POST /cases/{id}/sync` trigger.
    - Automatically queue high-priority referrals for background sync.

## Verification Instructions
- **Unit Check**: Bundle contains all 4 mandatory clinical artifacts.
- **Sync Check**: Case status in DB updates to `SYNCED` only after 200 OK from server.
- **Retry Check**: Simulate a 503 error on first 2 tries to verify backoff.
