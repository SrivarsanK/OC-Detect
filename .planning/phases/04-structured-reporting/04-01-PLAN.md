# Plan: 04-01 Clinical Reporting Engine (JSON/PDF)

## Requirements Covered
- **REPORT-01**: Structured clinical JSON generation.
- **REPORT-03**: PDF generation for clinical sharing.
- **REPORT-04**: Local encryption and storage.

## Goals
Implement a **ReportingService** to generate diagnostic "bundles" for each Case.

## Dependencies (Wave 0 - Parallelizable)
- `fpdf2` (Lightweight PDF support)
- Jinja2 (Schema-based JSON generation)

## Tasks (Wave 1)

1. **Structured JSON Generator**
    - Implement `ReportingService.generate_json(case_id: str)`
    - Format: OpenData standards (JSON-LD compatible)
    - Fields: Case ID, timestamp, triage_class, confidence, uncertainty, patient_id (None), model_version.
2. **Clinical PDF Template (fpdf2)**
    - Implement `ReportingService.generate_pdf(case_id: str)`
    - Design: "Diagnostic Summary" with:
        - Header: OralGuard Clinical Report (v1.0)
        - Section 1: Clinical Observation (Enhanced Image)
        - Section 2: AI Determination (Class + Confidence)
        - Section 3: Evidence Visualization (Grad-CAM Heatmap)
        - Section 4: Recommendation (Referral / Watch / Normal)
3. **Integration with Ingestion Flow**
    - Automatically trigger report generation at the end of `upload_image`.
    - Save files in `.oral_data/reports/json/` and `.oral_data/reports/pdf/`.
    - Update `Case` model with `report_path`.

## Verification Instructions
- **Unit Check**: Generate report for a known "malignant" case.
- **File Check**: Verify PDF opens and embeds images correctly.
- **Constraint Check**: Generation completes in <1s for Edge CPU.
