# Plan: 04-02 Diagnostic API Enhancement

## Requirements Covered
- **REPORT-02**: Integrated heatmap visual references.
- **SYNC-READY**: Diagnostic result bundling for Phase 5.

## Goals
Expand the ingestion API to return report metadata and provide endpoints to download the generated bundle.

## Dependencies (Wave 0)
- `ReportingService` (04-01-PLAN)

## Tasks (Wave 1)

1. **API: Download Endpoint**
    - Implement `GET /cases/{id}/report/pdf` (StreamingResponse).
    - Implement `GET /cases/{id}/report/json`.
2. **API: Case History**
    - Implement `GET /cases/` (Paginated list of triage outcomes).
    - Filter by: `referral=True`, `status=PROCESSED`.
3. **Database Migration**
    - Add `report_pdf_path` and `report_json_path` to `Case` model.
    - Ensure paths are returnable in common API schemas.

## Verification Instructions
- **Unit Check**: `GET /cases/{id}/report/pdf` returns a valid file.
- **Functional Check**: Verify the reporting path is correctly returned in the `upload` response.
- **Security Check**: Verify reports cannot be accessed without Case ID. (Basic v1)
