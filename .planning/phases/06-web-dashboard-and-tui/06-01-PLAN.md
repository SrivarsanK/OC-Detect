# Plan: 06-01 Dockerization & Backend Lifecycle

## Requirements Covered
- **DEPLOY-01**: Dockerize the OralGuard stack.

## Goals
Ensure the OralGuard backend is deployable and reproducible in a containerized environment (e.g. AWS/Azure/GCP).

## Dependencies (Wave 0)
- None.

## Tasks (Wave 1)

1. **Dockerfile (Multi-stage)**
    - Implement a `Dockerfile` for the FastAPI backend.
    - Optimized for slim size (Python 3.11/12-slim).
    - Install necessary libs (libgl1 for OpenCV).
2. **Environment Configuration**
    - Create a `.env.template` with all config options from `Settings`.
    - Map `STORAGE_DIR` to a persistent Volume in Docker.
3. **Docker Compose**
    - Orchestrate: `api-backend`, `redis` (optional for future queue), and `sqlite` volume.
    - Add a health check to ensure API is ready.

## Verification Instructions
- **Build Check**: `docker build -t oralguard-api .`
- **Compose Check**: `docker-compose up -d`
- **Volume Check**: Verify that `.oral_data` persists across container restarts.
